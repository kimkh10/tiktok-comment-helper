## 사용법

인자 없이 그냥 실행해도 된다 (이름과 슬랙 메시지를 자동으로 알아낸다). 이름이나 슬랙 메시지 원문을 직접 주면 그걸 우선 사용한다.

이 스킬은 `node`/로컬 파일에 의존하지 않는다. 파싱은 아래 규칙을 그대로 따라 직접(추론으로) 수행하고, 슬랙/브라우저 조작은 연결된 Slack 도구와 Claude for Chrome(브라우저 확장) 도구만 사용한다.

## 0단계: 이름과 슬랙 메시지 확보

1. **이름** — 사용자가 명시적으로 이름을 줬으면 그걸 쓴다. 안 줬으면 Slack의 "현재 사용자 프로필 조회" 도구(인자 없이 호출하면 로그인된 사용자 기준)로 Display Name/Real Name을 가져오고, `(` 앞부분만 잘라서 이름으로 쓴다 (예: "Anthony(김규현)" → "Anthony").
2. **슬랙 메시지** — 사용자가 원문을 직접 줬으면 그걸 쓴다. 안 줬으면:
   - Slack의 "채널 검색" 도구로 "틱톡_서포터즈" 채널을 찾는다 (채널 ID를 이미 알면 바로 써도 된다).
   - Slack의 "채널 메시지 읽기" 도구로 최근 메시지를 20~30개 정도 읽는다.
   - 오늘 날짜(한국 시간 기준, 메시지 타임스탬프로 판단)의 메시지만 추린다. 메시지 하나하나가 이미 "링크 + 번호 목록 + Made by" 블록 하나에 해당한다.
   - 오늘 날짜 메시지가 여러 개면 본문을 그대로 이어붙여 하나의 텍스트로 만든다. (슬랙 원문은 이름이 `*4. Anthony*`처럼 마크다운 굵게로 올 수 있으니, 아래 파싱 규칙에서 앞뒤 `*`는 이름의 일부가 아니라 서식으로 보고 제거한다.)

## 반드시 지킬 것

- **게시 버튼은 이 스킬이 절대 클릭하지 않는다.** 텍스트 입력까지만 하고 사람이 직접 확인하고 눌러야 한다.
- TikTok이 "Please wait..." 같은 봇 검증/캡차 화면을 보여주면 우회하지 않는다. 그 탭은 "차단됨"으로 기록하고 새로고침/재시도를 반복하지 않는다 (반복 조작이 오히려 검증을 더 유발할 수 있음이 테스트로 확인됨).
- 이미 동일한 텍스트의 댓글이 그 영상에 있으면 입력하지 않고 "이미 게시됨"으로 표시한다.
- 타이핑 직후 자바스크립트로 입력창 상태를 조회하지 않는다 — 포커스가 빠지면서 방금 입력한 내용이 사라지는 현상이 테스트로 확인됐다. 확인은 스크린샷으로만 한다.
- 같은 탭을 반복해서 새로고침/재시도하지 않는다. 항목당 한 탭, 한 번의 시도만 한다.

## 1단계: 텍스트 파싱 (직접 읽고 분석 — 외부 스크립트 없음)

주어진 슬랙 메시지 원문에서 다음 규칙으로 항목을 추출한다:

1. 텍스트 안에서 `https://www.tiktok.com/@계정이름/video/숫자` 형태의 링크를 순서대로 모두 찾는다. 각 링크가 하나의 "블록"을 시작한다. 블록의 범위는 그 링크 다음부터, 다음 링크가 나오기 전까지 또는 `Made by`가 포함된 마무리 줄(예: "— Made by anthony 고장 문의") 전까지다.
2. 각 블록 안에서 `숫자. 이름` 형식으로 시작하는 줄을 순서대로 찾는다 (예: "4. Anthony"). 그 줄 바로 다음부터 다음 번호 줄이 나오기 전까지의 텍스트가 그 사람의 댓글이다 (여러 줄이면 합쳐서 하나의 댓글로, 앞뒤 공백/빈 줄은 제거).
3. 사용자가 지정한 이름과 완전히 동일한 이름(대소문자 무시, 앞뒤 공백 제거, 부분 일치 금지)을 각 블록에서 찾는다:
   - 정확히 하나만 일치 → (링크, 댓글) 쌍을 게시 대상 큐에 추가.
   - 0개 또는 2개 이상 일치 → 그 블록은 "확인 필요"로 사용자에게 별도로 알리고 큐에서 제외한다. 절대 추측해서 넘겨짚지 않는다.
4. 큐에 있는 모든 (링크, 댓글) 쌍을 표로 정리해 사용자에게 보여준다.

## 2단계: 브라우저 도구 준비

Claude for Chrome 도구(예: 탭 컨텍스트 확인, 새 탭 열기, 페이지 이동, 스크린샷/클릭/타이핑, 페이지 텍스트 읽기에 해당하는 도구)를 사용한다. 도구가 바로 보이지 않으면 도구 검색으로 브라우저 관련 도구를 먼저 불러온다. 탭 그룹이 없으면 새로 만든다.

## 3단계: 큐에 있는 항목마다 새 탭을 하나씩 연다

각 (링크, 댓글)에 대해:

1. 새 탭을 만들고 해당 링크로 이동한다.
2. 스크린샷으로 로드 상태를 확인한다. 화면에 "Please wait" 같은 문구만 있고 실제 영상이 안 보이면 봇 검증 화면이다. 곧바로 포기하지 않고 10초 정도 대기한 뒤 같은 링크로 다시 이동해서 스크린샷으로 재확인한다. **최대 3번까지만** 재시도한다 (대기 시간을 10초→15초→20초 정도로 조금씩 늘린다). 3번째까지도 봇 검증 화면이면 "차단됨"으로 기록하고 이 탭은 그대로 둔 채 다음 항목으로 넘어간다 (그 이상 반복하지 않는다 — 과도한 반복이 오히려 검증을 더 유발할 수 있음이 테스트로 확인됨).
3. 오른쪽에 댓글 목록/입력창이 안 보이면, 영상 오른쪽의 댓글 아이콘(말풍선 모양)을 클릭해서 댓글 패널을 연다.
4. 페이지 텍스트를 읽어서, 게시하려는 댓글과 완전히 동일한 텍스트가 댓글 목록에 이미 있는지 확인한다. 있으면 "이미 게시됨"으로 기록하고, 이 탭을 닫은 뒤 다음 항목으로 넘어간다 (더 볼 필요가 없으니 열어둘 필요 없음).
5. 스크린샷을 보고 댓글 입력창의 위치를 파악해 클릭한다.
6. 댓글 텍스트를 실제 키 입력으로 타이핑한다.
7. **다시 스크린샷으로만** 텍스트가 입력창에 정상적으로 남아있는지 확인한다. 남아있으면 "입력 완료 - 게시 버튼 활성화 확인됨", 사라졌거나 잘 안 보이면 "입력 실패 - 직접 확인 필요"로 기록한다.
8. 게시 버튼은 누르지 않는다. 이 탭은 열어둔 채 다음 항목으로 넘어간다.

## 4단계: 최종 보고

각 항목의 상태(입력 완료 - 탭 열려있음 / 이미 게시됨 - 탭 닫음 / 차단됨 - 탭 열려있음 / 실패 - 탭 열려있음)를 표로 정리해서 사용자에게 보고하고, "입력 완료된 탭들에서 직접 확인 후 게시 버튼을 눌러달라. 이미 게시된 항목은 탭을 닫아뒀다"고 안내한다.
